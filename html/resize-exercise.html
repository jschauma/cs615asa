CS615A -- Aspects of System Administration

Resize Exercise

The objective of this exercise is for you to gain some more familiarty with the tools use to manipulate disks, partitions, and filesystems. The practical goal is to be able to resize an existing filesystem.

Perform the following tasks:

  • create a NetBSD instance
  • create a 1GB volume and attach it to the instance
  • use disklabel(8) to create two 512MB partitions on this volume
  • create a new filesystem on both partitions and mount them
  • create a file on one of the new filesystems
  • use resize_ffs(8) to expand the first partition over the second to a size of 1GB
  • verify that you can still access the file you created
  • now shrink the 1GB partition to 256MB
Then, repeat for a different OS, e.g., Linux. Note any differences in the process or tools. What happens to the filesystem structure, the number of inodes, the data blocks, the superblocks etc.?

You may find the following commands and manual pages helpful: aws ec2 create-volume, aws ec2 attach-volume, disklabel(8), mount(8), newfs(8), resize_ffs(8).

If you get stuck, check out this video for help. But it might be best for you to first try to solve the problem on your own.

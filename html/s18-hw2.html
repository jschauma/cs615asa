CS615A -- Aspects of System Administration

HW#2: Filesystem creation

Please carefully read this document in its entirety.

Summary:

The objective of this assignment is for you to set up a file system on an EBS volume and to verify the persistance of files created on this file system after an instance is terminated.

You will also practice how technical tasks like this can be captured, yielding an accurate history of the commands you used together with all of their input and output.

This assignment is worth 25 points.

Setting up and Using Cloud Storage

Perform the following tasks:

  • create two AWS instances of a given OS (Linux, FreeBSD, NetBSD, OmniOS)
  • create a 1GB volume and attach it to one of the instances
  • create a new filesystem on the volume and mount it
  • create a file on the new filesystem
  • terminate the first instance
  • attach the volume to the second instance
  • retrieve the file from the volume via the second instance
Then, repeat for a different OS. Note any differences in the process or tools. Can you use the same volume / filesystem across different OS instances? Why/how/why not?

You may find the following manual pages helpful: aws ec2 create-volume, aws ec2 attach-volume, format(1M), mkfs(8), mount(8), newfs(8).

Deliverables and Due Date

You will submit three plain text files. The first is a well-formatted, plain ascii text document (not an HTML document, PDF, Microsoft Word document, ...) README, in which you describe in plain (although properly spelled and well-formatted) English what you learned and what, if any, issues you ran into as well as how you overcame them.

The second file is a verbatim transcript of all the commands you ran for all of the above steps. This is most easily accomplished by using the script(1) utility. For example: 

$ ssh linux-lab.cs.stevens.edu
$ script -t 2>timestamps
Script started, file is typescript
$ aws ec2 run-instances --image-id AMI1234
...
$ ssh instance
remote$ commands here
remote$ exit
$ exit
Script done, file is typescript
$
You would then submit the files 'transcript' and 'timestamps' together with your README.
